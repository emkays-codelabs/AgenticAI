Core concept — Gen AI vs Agentic AI
Generative AI
Takes an input and generates output — text, images, SQL queries, etc. It responds but does not take action on your behalf. Example: ChatGPT writing a LinkedIn post, or a RAG system answering a question from documents.
Generates Responds No execution
Agentic AI
You give a goal. The system decides which tools or agents to invoke, executes tasks autonomously, and iterates in a loop until the goal is reached. An LLM sits at the center making decisions at every step.
Decides Executes Iterates
The autonomy scale
Agentic AI is not binary — it's a spectrum from low autonomy (a Python function in a feedback loop) to high autonomy (a fully self-driven multi-agent pipeline with no human involvement). "Human in the loop" sits in the middle: the system handles most steps but a person reviews or approves key decisions.
Key examples explained in class
Text-to-SQL agentic loop (low-to-medium autonomy)
The foundational agentic pattern shown in the lecture:
1
User writes a natural language question — e.g. "Show employees earning between 5,000 and 20,000."
2
LLM converts it to a SQL query automatically.
3
A Python function executes the query on the real database and captures the result or error.
4
If the query fails, the error trace is fed back to the LLM, which self-corrects and generates a new query.
5
This loop repeats until a valid query succeeds. The automatic retry loop is what makes it "agentic."
Multi-agent hiring pipeline (high autonomy)
Used to illustrate what high autonomy looks like with multiple specialised agents communicating:
1
HR uploads a job description to a chatbot interface.
2
Agent 1 parses the JD and auto-posts it to LinkedIn and Naukri via their APIs.
3
Agent 2 monitors applications for 7 days, collects all applicants, and stores them in a database.
4
Agent 3 pulls applications, parses resumes, matches them against the JD, and shortlists the top 3 candidates.
5
Agent 4 schedules interviews by accessing Google/Teams calendar and emailing candidates.
6
Agent 5 joins the interview, conducts the behavioural round, and produces a feedback report for HR.
7
Human in the loop for the final technical round — humans take over from here for the hiring decision.
Key concepts, protocols & frameworks
MCP — Model Context Protocol
Anthropic's communication protocol for agents. Converts a machine learning endpoint (e.g. a salary predictor API) into a conversational tool that any LLM can invoke via natural language.
ProtocolWill be covered
A2A — Agent-to-Agent Protocol
Google's emerging protocol for agents communicating directly with each other. Still evolving — will be added to the course once it stabilises.

LangChain
A framework (described as a "wrapper company") for building GenAI and agentic pipelines. Its newest addition is "Deep Agents." Will be used in the course.
FrameworkIn course
LangGraph
Built specifically for multi-agent, graph-based agentic systems. Treats agents as nodes and their interactions as edges. Used in the Travel Assistant and other projects.
FrameworkIn course
RAG — Retrieval Augmented Generation
You take data (PDFs, documents), embed it into a vector database, query it, feed results to an LLM, and the LLM generates an answer. The course will cover RAG with 500 GB+ datasets using Azure AI Search, plus RBAC (Role-Based Access Control) so users only receive answers from documents they're authorised to see.
Short-term vs long-term memory
Agents must decide whether to store intermediate results temporarily (short-term, in context) or persistently (long-term, in a database). This distinction is part of agent design patterns and will be covered in depth.

Teaching approach & philosophy
  • Start manual, then automate. Code will first be written from scratch in Python so concepts are clear, then tools like Cursor, Claude Code, or Windsurf are introduced to speed things up.
  • Frameworks are wrappers. LangChain, LangGraph, CrewAI, and n8n are convenience layers. Solid Python means you could build any of them yourself.
  • Code and resources shared after each session via the course dashboard or GitHub.
  • Hybrid cloud usage. Both AWS and Azure will be used. Free tiers are sufficient for most exercises; paid tiers may be needed for advanced projects.
  • Machine learning is still relevant. Traditional ML (random forest, etc.) often powers the tools that agents use — it's the backbone behind the scenes.
  • Practice is the only path. Building 3–4 production-grade projects is the minimum to develop real confidence for interviews and roles.
To prepare for next class
Make sure your Python fundamentals are solid — variables, functions, loops, conditionals, file handling. Also review basic OOP concepts (classes and objects). If you want a head start, read through the LangChain or LangGraph documentation and use an AI tool to help you navigate the content.
